# 利用規約・プロフィールページ 仕様書

## 1. 目的

- **利用規約ページ** (`/terms`): フッターにリンクがあるが未実装。法的要件を満たすページを作成する
- **プロフィールページ** (`/profile` または `/about` 拡充): ブログ運営者のプロフィールを掲載するページ

## 2. 利用規約ページ (`/terms`)

### 2.1 URL
`/terms`

### 2.2 掲載内容
1. 適用範囲
2. 禁止事項
3. 免責事項
4. 著作権
5. リンクについて
6. プライバシーポリシーへの参照
7. 規約の変更
8. お問い合わせ

### 2.3 デザイン要件
- プライバシーポリシーページ (`/privacy`) と同一のスタイルを踏襲
- `metadata` に title・description・canonical を設定
- 制定日・最終改定日を表記

### 2.4 メタデータ
```tsx
export const metadata: Metadata = {
  title: "利用規約",
  description: "おひとりさまライフの利用規約をご確認ください。",
  alternates: { canonical: "/terms" },
};
```

---

## 3. プロフィールページ (`/profile`)

### 3.1 URL
`/profile`

### 3.2 掲載内容
1. **Hero**: 運営者名・キャッチコピー
2. **自己紹介**: テキストと画像（about ページのパターン）
3. **経歴・バックグラウンド**: 箇条書きまたはタイムライン
4. **ブログについて**: このブログを始めた理由・目的
5. **SNS・外部リンク**: お問い合わせへの導線

### 3.3 デザイン要件
- About ページ (`/about`) のコンポーネント構成を参考に
- Material Design 3 カラートークンを使用
- レスポンシブ対応（モバイルファースト）

### 3.4 メタデータ
```tsx
export const metadata: Metadata = {
  title: "プロフィール",
  description: "おひとりさまライフの運営者プロフィールです。",
  alternates: { canonical: "/profile" },
};
```

---

## 4. 共通要件

- Server Component（`'use client'` 不要）
- `app/(public)/` 配下に配置
- `page.test.tsx` によるテストを作成
- フッターリンクとの整合性確認（`/terms` はすでにリンクあり）

## 5. 成功基準

- [ ] `/terms` にアクセスして利用規約が表示される
- [ ] `/profile` にアクセスしてプロフィールが表示される
- [ ] 両ページのテストが通る
- [ ] メタデータ（title・canonical）が正しく設定されている
- [ ] フッターの `/terms` リンクが機能している
