# ニュースレター購読機能 仕様書

## 1. 目的・背景

ブログ読者がメールアドレスを登録することで、新着記事・厳選コンテンツを定期的に受け取れるようにする。
UIコンポーネント（NewsletterSignup.tsx、page.tsx の NewsletterSection）は既に実装済みだが、バックエンド処理・フォーム送信ロジックが未実装の状態。

## 2. 機能要件

### 2.1 購読登録フロー

1. ユーザーがメールアドレスを入力して「登録する」ボタンを押す
2. フロントエンドがメールアドレスを `/api/newsletter/subscribe` へ POST
3. APIがメールアドレスをバリデーション（形式チェック・重複チェック）
4. バリデーション通過後、購読者データを Vercel Blob に保存
5. 登録確認メールをユーザーへ送信
6. フロントエンドに成功/失敗を返却
7. UIに結果を表示（成功メッセージ or エラーメッセージ）

### 2.2 エラーケース

| エラーケース | HTTPステータス | ユーザー向けメッセージ |
|------------|--------------|---------------------|
| メール形式不正 | 400 | 「有効なメールアドレスを入力してください」 |
| すでに登録済み | 409 | 「このメールアドレスはすでに登録されています」 |
| サーバーエラー | 500 | 「登録に失敗しました。しばらく後にお試しください」 |

### 2.3 登録確認メール

- 件名: 「ニュースレター登録が完了しました」
- 内容: 登録完了の確認 + サイトへの誘導リンク
- 送信元: Gmail SMTP（既存の GMAIL_USER / GMAIL_APP_PASSWORD を流用）

## 3. 非機能要件

- **レスポンス時間**: APIレスポンス 3秒以内
- **セキュリティ**: メールアドレスのサニタイズ、レート制限は将来対応
- **テスト**: ユニットテスト（vitest）でカバレッジ 80% 以上

## 4. データ設計

### 購読者データ（Vercel Blob: `newsletter/subscribers.json`）

```json
{
  "updatedAt": "2026-03-31T00:00:00.000Z",
  "subscribers": [
    {
      "id": "uuid-v4",
      "email": "user@example.com",
      "subscribedAt": "2026-03-31T00:00:00.000Z"
    }
  ]
}
```

### 型定義

```typescript
interface Subscriber {
  id: string;          // UUID v4
  email: string;       // メールアドレス
  subscribedAt: string; // ISO 8601
}

interface SubscribersData {
  updatedAt: string;
  subscribers: Subscriber[];
}
```

## 5. APIインターフェース

### POST /api/newsletter/subscribe

**リクエスト**:
```json
{
  "email": "user@example.com"
}
```

**レスポンス（成功）**:
```json
{
  "success": true,
  "message": "登録が完了しました。確認メールをお送りしました。"
}
```

**レスポンス（エラー）**:
```json
{
  "success": false,
  "error": "このメールアドレスはすでに登録されています"
}
```

## 6. UIの変更

### NewsletterSignup.tsx（`components/layout/Sidebar/`）

- `onSubmit` ハンドラーを実装
- 送信中: ボタンを無効化 + ローディング表示
- 成功: 「登録が完了しました！」メッセージを表示
- エラー: エラーメッセージをフォーム下部に表示

### NewsletterSection（`app/(public)/page.tsx`）

- 同様のロジックを実装（または共通フックで共有）
