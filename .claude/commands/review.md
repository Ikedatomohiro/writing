# /review コマンド

コード変更をレビューします。

## 使用方法

```
/review           # 全ての変更をレビュー
/review [ファイル] # 特定ファイルをレビュー
```

## 実行内容

1. `git diff` で変更を取得
2. `.claude/agents/code-reviewer.md` のガイドラインに従ってレビュー
3. 重大度別にフィードバックを整理
4. 承認ステータスを判定

## レビュー観点

- **セキュリティ**: 脆弱性、認証情報の露出
- **コード品質**: 関数サイズ、ネスト深度、エラーハンドリング
- **パフォーマンス**: アルゴリズム効率、クエリ最適化
- **ベストプラクティス**: 命名規則、一貫性

## 出力形式

```markdown
## レビュー結果

### Critical（修正必須）
- [ファイル:行番号] 問題の説明

### High（強く推奨）
- ...

### Medium（推奨）
- ...

## 承認ステータス
[承認 / 要修正]
```
