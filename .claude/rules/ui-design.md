# UI Design Rules

## デザインツール

UIデザインの作成・編集には、以下のツールを目的に応じて使い分けること。

### 1. Figma MCP サーバー（推奨）

**デザインシステムに基づく構造的なデザインを作成する場合に使用する。**

Figma MCPサーバーの `use_figma` ツールにより、AIエージェントがFigmaキャンバス上で直接デザインを作成・編集できる。既存のコンポーネント・変数・トークンを再利用した構造的なデザイン生成が可能。

**対象作業**:
- デザインシステムを活用した画面設計
- コンポーネントセット・バリアントの作成
- デザイントークン（変数）の管理
- Auto Layoutを使った構造的なレイアウト

**ツールの使い分け**:
- `use_figma`: 構造的デザイン生成（コンポーネント、変数、Auto Layout）。**デザインシステムとの接続あり**
- `generate_figma_design`: コードからの高速モックアップ。ライブラリ接続なし

**重要な原則**:
- 新規作成の前に、必ず既存のコンポーネント・変数を確認し再利用する
- フラットなレイヤーではなく、Auto Layout・コンポーネント・バリアントを使った構造を生成する
- デザイントークンをハードコードせず、Figma変数として管理する

### 2. pencil MCPサーバー

**Figma MCPサーバーが利用できない場合の代替手段。**

#### pencilツールの使い方

1. **エディタ状態の確認**: `get_editor_state`で現在の状態を確認
2. **ドキュメントを開く**: `open_document`で.penファイルを開くか新規作成
3. **ガイドライン取得**: `get_guidelines`でデザインルールを確認
4. **デザイン作成**: `batch_design`でUI要素を配置
5. **検証**: `get_screenshot`で結果を確認

#### ファイル配置

.penファイルは`design/`ディレクトリに配置する。

```
design/
  user-page.pen
  dashboard.pen
  components.pen
```

## デザインエージェントの使用

UIデザインに関するタスクでは、`.claude/agents/designer.md` に定義されたデザインエージェントが使用される。デザインエージェントは以下を担当する:

- デザインシステムに基づくUI設計
- Figma MCPツールを活用したキャンバス操作
- デザインとコード（Tailwind CSS v4）の一貫性担保

## 禁止事項

- HTMLやCSSで直接UIをデザインすること（実装フェーズは除く）
- .penファイルを`Read`や`Grep`ツールで直接読むこと（暗号化されているため`batch_get`を使用）
- デザイントークンを使わずにハードコードされた値でデザインすること

## 共通ワークフロー

1. 要件を確認
2. 既存デザインシステム（コンポーネント、変数、トークン）を確認
3. デザインツールでUIを構築
4. スクリーンショットで視覚的に検証
5. 必要に応じて調整を繰り返す
6. Tailwind CSS v4での実装可能性を確認
