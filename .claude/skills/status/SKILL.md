# Status Check

プロジェクトの実装状態を確認し、次のアクションを提案する。

## 確認手順

### 1. PLAN.mdの確認

プロジェクトルートのPLAN.mdを読み、以下を把握:

- 全体のフェーズ構成
- 各フェーズのタスク一覧
- 完了済み（`[x]`）と未完了（`[ ]`）のタスク
- 現在アクティブなフェーズの特定

### 2. 対象機能のspec.mdを探す

現在のフェーズに関連するspec.mdを検索:

```bash
# designディレクトリ内を検索
find design -name "spec.md" 2>/dev/null

# 機能名で検索
find . -path "*/機能名/spec.md" 2>/dev/null
```

spec.mdが見つかった場合:
- 要件一覧を確認
- 成功基準を確認
- 制約事項を確認

### 3. 実装ディレクトリの状態確認

PLAN.mdの成果物セクションに記載されたディレクトリ・ファイルの存在を確認:

```bash
# ファイル存在チェック
ls -la app/対象ディレクトリ/
ls -la lib/対象ディレクトリ/
ls -la components/対象ディレクトリ/
```

### 4. task.mdの進捗確認

task.mdが存在する場合、タスクの完了状況を確認:

- 完了済みタスク数
- 進行中タスク
- 未着手タスク
- ブロッカーの有無

### 5. Gitブランチ状態

```bash
git branch --show-current
git status --short
```

## 出力フォーマット

```markdown
## Status Report

### 現在のフェーズ
- フェーズ: [Phase X: 名前]
- 進捗: [完了タスク]/[全タスク]

### spec.md
- 状態: 存在する / 存在しない
- パス: [パス]
- 要件数: [数]

### 実装状況
| ファイル/ディレクトリ | 状態 |
|----------------------|------|
| app/xxx | 存在 / 未作成 |
| lib/xxx | 存在 / 未作成 |

### task.md
- 状態: 存在する / 存在しない
- 完了: [数]/[数]

### Gitブランチ
- ブランチ: [ブランチ名]
- 未コミット変更: あり / なし

### 次のアクション
1. [具体的なタスク]
2. [具体的なタスク]
```
