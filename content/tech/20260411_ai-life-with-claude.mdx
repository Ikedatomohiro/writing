# AIと生きる、という選択

正直、最初は「楽になるだけ」だと思ってたんだよね。

Claude Code を使い始めたのは去年の夏ごろ。IT転職してから3年、ようやく現場の感覚がつかめてきたタイミングだった。「AIツールで作業が速くなる、それだけのことでしょ」って、そのくらいの気持ちで触り始めた。

でも実際に使い続けていくうちに、気づいたらそれだけじゃない変化が起きてたんだよね。

---

## 「コードを書く」から「コードを考える」へ

転職直後の僕は、毎日ひたすら手を動かしてた。コードを書いて、エラーを見て、Stack Overflowを検索して、また書いて。そのサイクルが「仕事してる感」だったんだと思う。

Claude Code を使い始めてから、そのサイクルが変わった。

たとえばAPIのエンドポイントを新規で作るとき、以前は「まず雛形を書いて、認証どうしよう、エラーハンドリングはどこで……」ってパズルを一人で解いていた。今は「こういう要件で、こういう制約があって、ここがトリッキーなんだけど」って会話するところから始めるんだよね。

作業のスタート地点が変わった感じ。

最初のうちは「AIに聞けばすぐ答えが出る」という感覚で使ってたんだけど、しばらくすると「もっとうまく質問を作れば、もっといい答えが返ってくる」ってことに気づいてしまった。

そこから、コードを「書く」より「考える」時間が増えていった。

---

## 36歳エンジニアが感じた、ちょっとした焦りと開放感

IT転職組の僕にとって、若い同僚のコーディングスピードは正直プレッシャーだったんだよね。

「あの子は20代から書いてるから指が違う」「経験年数で差がある」……そういう言い訳を頭のどこかに置いてた気がする。

Claude Code を使い始めてから、そのプレッシャーの種類が変わったかもしれない。。

速さじゃなくて「どう設計するか」「どう問題を定義するか」のほうが問われるようになってきた感覚がある。これは僕みたいな転職組にとっては、案外チャンスかもしれない。現場の経験から積んできた「なぜこの設計になってるか」「この仕様には背景がある」みたいな文脈理解は、AIにそのままは渡せない部分だから。

ただ、焦りもある。

「AIが使いこなせる人」と「AIを使ってる気になってる人」の差が、これからどんどん開いていく気がして。後者にならないように、という緊張感はずっとある。

---

## 具体的に変わった、日常の仕事フロー

抽象的な話ばかりしてても仕方ないから、実際のフロー変化を書いてみる。

### 以前の1日

- 朝イチでタスク確認 → コードエディタを開く
- ひたすら実装 → 詰まったら検索 → 実装
- レビュー前に軽くリファクタ → PR提出

### 今の1日

- 朝イチでタスク確認 → Claude Code に「今日これをやる、現状のコードはこう、懸念点はここ」と伝える
- 設計方針を対話で整理 → 実装に入る
- 実装中に「ここ迷ってる」を逐次相談 → ブロッカーを素早く解消
- PR前に「このコード、見落としてそうな点ある？」と最終確認

作業の密度が変わったというか、詰まる時間が減った分、別のことを考える余裕ができてきた感じ。

---

## 「使いこなす」の定義が変わってきた

Claude Code を使い始めた頃、「うまく使えた」って感じる瞬間は「自分では書けないコードを出力してくれたとき」だった。

今は少し違う。

「自分の考えを整理する助けになったとき」がいちばん手応えある。

実装の迷いを言語化してClaude に話しかけると、自分の中でモヤっとしていたものが、会話の中でクリアになっていく。あの感覚は、AI側がすごいというよりも、「話しながら考える」という行為自体が効くんだと思う。。

ツールとしての優秀さはもちろんある。でも僕にとっての価値は「自分の思考を引き出してくれる」ところにあって、それはちょっとだけ、ペアプログラミングのパートナーみたいな感覚に近いかもしれない。

---

## AIと生きるって、どういうことか

「AIと生きる」という言葉、最初は大げさに聞こえてたんだよね。

でも毎日使ってると、これって働き方の話だけじゃない気がしてきた。

問題の立て方、会話の仕方、自分の考えの整理の仕方——そういうところが、少しずつ変わってきている。楽になった部分もある。でもそれ以上に、「変わった」って感じのほうが強い。

楽になるのを求めてたつもりが、結果として「どう考えるか」を問われる場面が増えた。これが良いことなのか、まだ正直わかってない部分もある。。

でも、毎日使い続けてるのはたしかだし、使わない選択肢は今のところ思い浮かばない。

それが、今の僕にとっての「AIと生きる」ということかもしれない。

---

AIとの付き合い方は、たぶんこれからも変わり続けると思う。

今日のこの記事も、ある意味そのプロセスの途中だ。答えを持ってるわけじゃないけど、迷いながら考え続けることが、今いちばん正直なスタンスだと思ってる。
