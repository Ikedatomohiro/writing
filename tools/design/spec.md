# 画像選定・生成機能 仕様書

## 目的・背景

writerエージェントが生成する記事にはテキストしか含まれておらず、視覚的な魅力が不足している。記事内容に合った画像を自動で選定・提案する機能を追加し、記事の品質を向上させる。

## 機能要件

### 1. 画像提案ノード（ImageSuggestionNode）

記事の統合結果（IntegratorNode出力後）を受け取り、以下の画像を提案する。

#### 1.1 アイキャッチ画像
- 記事のサムネイルとして使用する画像を1枚提案
- 記事タイトル・キーワードから適切な検索クエリを生成
- Unsplash APIで検索し、最適な画像URLを返す

#### 1.2 本文挿入画像
- 記事の各セクションに適した画像を提案（0〜3枚）
- セクション内容に基づいた検索クエリを生成
- 必要に応じて図解の説明テキストも提案

#### 1.3 OGP画像
- SNSシェア用の画像を提案
- アイキャッチ画像と兼用可能

### 2. 画像取得方法

| 優先度 | 方法 | API | 用途 |
|--------|------|-----|------|
| 1（本実装） | ストック検索 | Unsplash API | 写真素材 |
| 2（将来） | AI生成 | DALL-E等 | オリジナル画像 |

### 3. 入出力

#### 入力
- 記事タイトル（WriterOutput.title）
- 記事本文（WriterOutput.content）
- キーワード（WriterInput.keywords）
- セクション情報（WriterOutput.sections）

#### 出力
- ImageSuggestions: 画像提案のリスト
  - eyecatch: アイキャッチ画像情報
  - inline_images: 本文挿入画像リスト
  - ogp: OGP画像情報

### 4. Unsplash APIクライアント

- 環境変数 `UNSPLASH_ACCESS_KEY` からAPIキーを取得
- 検索エンドポイント: `GET /search/photos`
- レスポンスから画像URL、撮影者情報、ライセンス情報を抽出
- APIキー未設定時はスキップ（フォールバック）

## 非機能要件

- APIキー未設定時もエラーにならず、画像なしで動作する
- 著作権: Unsplash License（フリー素材）のみ使用
- API呼び出し回数: 1記事あたり最大5回
- タイムアウト: 1リクエスト10秒

## 制約事項

- AI生成画像は本実装の対象外（将来拡張）
- 画像のダウンロード・保存は対象外（URLのみ提案）
- Unsplash API Rate Limit: 50 requests/hour（Demo）

## 成功基準

1. writerエージェントの既存テストが全てパスする
2. 画像提案ノードの単体テストが全てパスする
3. APIキー未設定時でもエージェント全体が正常動作する
4. WriterOutputに画像提案情報が含まれる
